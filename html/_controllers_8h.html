◆ control_set_delay()

void control_set_delay ( ubyte  delay)

Set the delay used to loop all controllers; By default, this is set to 20. Lowering it is pointless as motors update at this frequency. Higher delays don't actually have that much of a difference if slewing is on.

Parameters
delayDelay, in milliseconds, between each call of the controller. Set to 20 ms by default.

void control_switch ( controller_t  task_code)

Switch to a controller that is a parameter-less function of type void. This function will be internally looped by taskDelayUntil. If no controller is running, this will immediately start the function. The thread handle for this is managed internally.

◆ control_xdrivecorner()

void control_xdrivecorner ( )

Control chassis via an xdrive configuration where the corners are the wheels. Left joystick is for movement, right for turning. This assumes that, when all motors are powered with a positive value via the Motor module, the robot turns clockwise. This will try to slew motors if possible.

◆ control_xdrivecorner_gamer()

void control_xdrivecorner_gamer ( )

Control chassis via an xdrive configuration where the corners are the wheels. Left joystick is for forward / backward, right for turning; Kind of like a racing game. This assumes that, when all motors are powered with a positive value via the Motor module, the robot turns clockwise. This will try to slew motors if possible.

◆ control_xdrivecorner_serial()

void control_xdrivecorner_serial ( )

Control chassis via an xdrive configuration where the corners are the wheels. Left joystick is for movement, right for turning. This uses serial input via the serial module and assumes that the controller input of remoteTransmitter.py matches that of the VEXnet controller. This will try to slew motors if possible.

◆ control_xdriveedge()

void control_xdriveedge ( )

Control chassis via an xdrive configuration where the edges are the wheels; The "front" of the robot is a wheel. Left joystick is for movement, right for turning. This assumes that, when all motors are powered with a positive value via the Motor module, the robot turns clockwise. This will try to slew motors if possible.
